# NAS Drive 서비스 개요

## Lean Canvas

| 블록 | 내용 |
|---|---|
| **문제** | 1. 상용 클라우드(Google Drive 등)에 개인 데이터를 맡기는 것에 대한 프라이버시/비용 우려 2. NAS 기본 파일 관리 UI가 불편하고 클라우드 수준의 UX를 제공하지 못함 3. 가족/친구와 파일을 간편하게 공유할 방법이 부족 |
| **고객 세그먼트** | 개인/가족/친구 단위 소규모 그룹 (1~5명), NAS를 보유하거나 셀프호스팅에 관심 있는 사용자 |
| **고유 가치 제안** | 내 NAS에서 돌아가는 Google Drive — 데이터 주권을 지키면서 클라우드급 파일 관리 UX 제공 |
| **솔루션** | 웹 기반 파일 관리(업로드/다운로드/폴더/검색/즐겨찾기/휴지통) + 공유 링크 + 초대 기반 멀티유저 |
| **채널** | 자체 NAS Docker 배포, GitHub 오픈소스 |
| **수익원** | 없음 (개인 프로젝트/오픈소스) |
| **비용구조** | NAS 하드웨어, 도메인/SSL, 개발 시간 |
| **핵심지표** | 업로드/다운로드 성공률, 응답 시간, 스토리지 사용량, 활성 사용자 수 |
| **경쟁우위** | 완전한 데이터 소유권, 커스터마이징 자유, 외부 의존성 최소화 (소셜 로그인 없음), NAS Media 마이크로서비스 확장 구조 |

## 경쟁 서비스 벤치마킹

| 항목 | Google Drive | Synology Drive | Nextcloud | **NAS Drive (우리)** |
|---|---|---|---|---|
| 호스팅 | 클라우드 | NAS 전용 | 셀프호스팅 | 셀프호스팅 (NAS) |
| 파일 관리 | ★★★★★ | ★★★★ | ★★★★ | ★★★ (목표) |
| 공유 | 링크 + 권한 | 링크 + 권한 | 링크 + 권한 | 링크 + 권한 |
| 미디어 미리보기 | 내장 | 내장 | 플러그인 | NAS Media 연동 |
| 모바일 | 네이티브 앱 | 네이티브 앱 | 앱 존재 | v0.2 목표 |
| 기술 자유도 | 없음 | 제한적 | 높음 (PHP) | 최대 (오픈소스) |
| 데이터 주권 | ✕ | ○ | ○ | ○ |
