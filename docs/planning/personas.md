# NAS Drive 페르소나

## Persona 1: 민수 (서비스 운영자)

| 항목 | 내용 |
|---|---|
| **역할** | NAS 소유자, 서비스 관리자 |
| **목표** | 가족/친구를 위한 개인 클라우드 구축 및 운영 |
| **기술 수준** | 개발자 (Docker, 서버 관리 가능) |
| **Pain Point** | 상용 서비스의 용량 제한과 비용, Synology 기본 UI의 한계 |
| **핵심 시나리오** | 서비스 배포, 사용자 초대, 스토리지 모니터링, 시스템 관리 |

## Persona 2: 지은 (일반 사용자)

| 항목 | 내용 |
|---|---|
| **역할** | 민수의 가족/친구, 초대받은 사용자 |
| **목표** | 간편하게 파일 업로드/다운로드, 공유 링크로 파일 전달 |
| **기술 수준** | 일반 (Google Drive 사용 경험 있음) |
| **Pain Point** | 복잡한 NAS 접속 방법, 클라우드처럼 직관적인 UI 부재 |
| **핵심 시나리오** | URL 접속 → 로그인 → 파일 업로드 → 공유 링크 생성 → 지인에게 전달 |

## Persona 3: 비로그인 방문자

| 항목 | 내용 |
|---|---|
| **역할** | 공유 링크를 받은 외부인 |
| **목표** | 링크 클릭 → 파일 다운로드 (가입/로그인 없이) |
| **기술 수준** | 무관 |
| **핵심 시나리오** | 공유 링크 접속 → 파일 확인 → 다운로드 |
