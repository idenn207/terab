# A-10 시스템 모니터링

## 화면 개요

- 목적: NAS 서버의 CPU, 메모리, 디스크 사용량을 실시간으로 모니터링
- 진입 경로: `/system/monitor` (사이드바 "시스템" → 서브메뉴 "모니터링")
- 사용자: `system:monitor` 권한 보유 관리자 (페르소나: 민수)
- 레이아웃: **Layout-B** (Admin Web 기본)
- 플랫폼: PC Web 전용
- 관련 요구사항: ADMIN-11

## 인터랙션 상세

| # | 사용자 액션 | 시스템 반응 |
|---|-------------|-------------|
| 1 | 페이지 진입 | 현재 상태 데이터 로드 + 추이 차트 데이터 로드 + 선택된 주기로 폴링 시작 |
| 2 | 갱신 주기 변경 | 기존 폴링 중단 → 새 주기로 폴링 재시작 |
| 3 | 새로고침 클릭 | 즉시 데이터 갱신 (폴링 타이머 리셋) |
| 4 | 시간 범위 탭 변경 | 해당 기간의 추이 데이터 로드, 차트 갱신 |
| 5 | 범례 시리즈 클릭 | 해당 시리즈 차트에서 숨김/표시 토글 |
| 6 | 차트 위 마우스 호버 | 툴팁 표시 (시각 + CPU/메모리/디스크 수치) |
| 7 | 브라우저 탭 비활성화 | Page Visibility API로 폴링 일시정지, 상태 표시등 🟡 |
| 8 | 브라우저 탭 다시 활성화 | 즉시 데이터 갱신 + 폴링 재시작, 상태 표시등 🟢 |
| 9 | 페이지 이탈 | 폴링 타이머 클리어 (리소스 정리) |

### 폴링 동작

- 선택된 주기(기본 10초)마다 `/api/admin/system/metrics` 엔드포인트 호출
- 응답 실패 시 3회 재시도 후 상태 표시등 🔴 + 에러 상태 전환
- 재시도 간격: 2초 → 4초 → 8초 (지수 백오프)
- Page Visibility API (`document.hidden`)로 탭 비활성 감지 시 폴링 중단

## 접근성

- 게이지 차트: `role="meter"`, `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`, `aria-label` (예: "CPU 사용률 45%")
- 라인 차트: `role="img"`, `aria-label` (현재 수치 요약 텍스트)
- 폴링 상태 변경: `aria-live="polite"` (연결 실패, 일시정지 등 상태 변경 안내)
- 갱신 주기 드롭다운: `aria-label="데이터 갱신 주기"`
- 시간 범위 탭: `role="tablist"`, `role="tab"`, `aria-selected`
- 색상 외 텍스트 수치 병행 (게이지 색상만으로 상태 판단하지 않도록)
