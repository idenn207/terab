# D-01 로그인 — 1단계: 자격증명 입력

## 화면 개요

- 목적: 사용자 ID/PW 검증 (1차 인증)
- 진입 경로: `/login` (미인증 상태에서 모든 URL 접근 시 리다이렉트)
- 사용자: 등록된 모든 사용자 (페르소나: 민수, 지은)
- 레이아웃: **Layout-C** (인증 페이지)
- 플랫폼: Web + 모바일 앱 공용

## 구성 요소

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 서비스 로고 | 이미지/텍스트 | "NAS Drive" 서비스 로고 | - |
| 아이디 입력 | Input (text) | 사용자 ID 입력 | 기본 / 포커스 / 에러(빨간 테두리 + 메시지) |
| 비밀번호 입력 | Input (password) | 비밀번호 입력, 우측 표시/숨김 토글 | 기본 / 포커스 / 에러 |
| 비밀번호 표시 토글 | IconButton | 👁 비밀번호 평문 표시/마스킹 전환 | 숨김(기본) / 표시 |
| 이 기기를 신뢰 | Checkbox | 체크 시 30일간 2FA 스킵 (PC Web에서만 표시) | 미체크(기본) / 체크 |
| 로그인 버튼 | Button (primary) | 1차 인증 요청 | 기본 / 호버 / 로딩(스피너) / 비활성(입력 미완료) |
| 에러 메시지 | Alert | 인증 실패 시 상단에 표시 | "아이디 또는 비밀번호가 올바르지 않습니다" |

## 인터랙션

| 사용자 액션 | 시스템 반응 |
|-------------|-------------|
| ID/PW 입력 후 로그인 클릭 | API 호출 (`POST /auth/login`) |
| 1차 인증 성공 + 신뢰기기 | JWT 발급 → `/drive`로 이동 (2FA 스킵) |
| 1차 인증 성공 + 비신뢰기기 | **D-01a (2FA 대기 화면)**으로 전환 |
| 1차 인증 실패 | 에러 메시지 표시, 비밀번호 필드 초기화 |
| 유효한 토큰 보유 상태로 접근 | `/drive`로 자동 리다이렉트 |
| Enter 키 입력 | 로그인 버튼 클릭과 동일 |
| 입력값 미완료 상태 | 로그인 버튼 비활성 |
| **모바일 앱:** 생체인증 활성화 상태 | 지문/Face ID → Refresh Token 검증 → `/drive` 이동 (ID/PW, 2FA 모두 스킵) |

## 반응형 동작

- 데스크톱: 카드 400px, 화면 중앙 정렬
- 모바일 앱: 전체 화면, 생체인증 버튼 하단에 추가 표시
