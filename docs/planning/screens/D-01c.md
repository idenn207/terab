# D-01c 2FA 승인 (모바일 앱)

## 화면 개요

- 목적: PC 로그인 요청에 대해 모바일에서 숫자 매칭 승인/거부
- 진입 경로: Push 알림 탭 또는 앱 실행 시 대기 챌린지 자동 감지
- 사용자: 등록된 디바이스를 가진 사용자
- 레이아웃: 전체 화면 모달 (앱 내)
- 플랫폼: **모바일 앱 전용**

## 구성 요소

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 제목 | Text (h2) | "로그인 요청" | - |
| 요청 정보 | Text | 접속 IP, 시간 등 맥락 정보 | - |
| 안내 문구 | Text | "PC에 표시된 숫자를 입력하세요" | - |
| 숫자 입력 | Input (number, 2자리) | 2개의 개별 숫자 입력 박스 | 기본 / 포커스 / 에러 |
| 승인 버튼 | Button (primary) | 숫자 매칭 + 승인 전송 | 기본 / 호버 / 로딩 / 비활성(숫자 미입력) |
| 거부 버튼 | Button (danger, outline) | 로그인 거부 전송 | 기본 / 호버 |
| 남은 시간 | Timer | 60초 카운트다운 | 진행 중 / 만료 |

## 인터랙션

| 사용자 액션 | 시스템 반응 |
|-------------|-------------|
| Push 알림 탭 | 앱 열림 → 2FA 승인 화면 자동 표시 |
| 앱 직접 실행 (Push 꺼짐) | `GET /auth/2fa/pending` 폴링 → 대기 챌린지 존재 시 승인 화면 자동 이동 |
| 올바른 숫자 입력 + 승인 | `POST /auth/2fa/verify { action: "approve" }` → PC에 JWT 전달 |
| 잘못된 숫자 입력 + 승인 | 에러: "숫자가 일치하지 않습니다" |
| 거부 클릭 | `POST /auth/2fa/verify { action: "deny" }` → PC에 거부 통보 |
| 타임아웃 (60초) | "인증 시간이 만료되었습니다" → 화면 닫힘 |

## Push 알림 비활성화 시 동작

- 앱 foreground 진입 시 `PushNotifications.checkPermissions()` 확인
- Push 꺼져 있으면 상단 배너: "알림을 켜면 로그인 승인이 더 빠릅니다" + [알림 설정 열기]
- 동시에 `GET /auth/2fa/pending`으로 대기 챌린지 확인 → 존재 시 즉시 승인 화면 표시
