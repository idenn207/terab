# D-04 폴더 탐색

## 화면 개요

- 목적: 특정 폴더 내 파일/폴더 목록 조회 및 관리
- 진입 경로: `/drive/:folderId` (D-03 또는 D-04에서 폴더 더블클릭/이름 클릭)
- 사용자: 로그인된 모든 사용자 (페르소나: 민수, 지은)
- 레이아웃: **Layout-A** (Drive Web 기본)
- 플랫폼: Web + 모바일 앱 공용
- 기본 동작: **D-03과 동일** — 아래 차이점만 기술

## D-03 대비 차이점

### 1. 브레드크럼 변경

```
┌─ Toolbar ──────────────────────────────────────────────────┐
│ 📁 내 드라이브 > 사진 > 2026년 3월    정렬▼  ☷ ≡  [+ 업로드] [📁 새 폴더] │
└────────────────────────────────────────────────────────────┘
```

- 각 경로 세그먼트가 클릭 가능 (해당 폴더로 이동)
- "내 드라이브" 클릭 → `/drive` (D-03)으로 이동
- 경로가 3단계 이상일 때: 중간 세그먼트를 "..." 드롭다운으로 축약

```
📁 내 드라이브 > ... > 프로젝트 > NAS > src
                  ↓
            ┌──────────┐
            │ 사진     │
            │ 문서     │
            │ 프로젝트 │  ← 중간 경로 펼침
            └──────────┘
```

### 2. URL 구조

- `/drive/:folderId` — folderId는 서버 발급 고유 식별자

### 3. 빈 폴더 상태

- D-03 "드라이브가 비어 있습니다" → **"이 폴더는 비어 있습니다"** 로 메시지 변경
- 동일하게 [업로드] [새 폴더] CTA 버튼 표시

### 4. 상위 폴더 이동

- 브레드크럼의 상위 경로 클릭으로 이동
- 모바일 앱: 시스템 뒤로가기 제스처로 상위 폴더 이동 지원

### 5. 그 외

- 구성 요소, 인터랙션, 모달, 상태, 반응형, 접근성 → **D-03 참조**
