# A-04 사용자 상세

## 화면 개요

- 목적: 개별 사용자의 상세 정보 조회, 역할 할당/해제, 비활성화/삭제 관리
- 진입 경로: `/users/:id` (A-03 사용자 목록에서 행 클릭)
- 사용자: `user:read` + `user:role` 권한 보유 관리자 (페르소나: 민수)
- 레이아웃: **Layout-B** (Admin Web 기본)
- 플랫폼: PC Web 전용
- 관련 요구사항: ADMIN-03, ADMIN-04, ADMIN-05
- 관련 유저플로우: Flow 6 — 사용자 상세 > 역할 할당 / 비활성화 / 삭제

## 구성 요소

### Page Header

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 뒤로가기 | Link | "← 사용자 목록" → A-03으로 이동 | 기본 |
| 사용자 이름 | Text (h1) | 해당 사용자 닉네임 | - |
| 비활성화 버튼 | Button (secondary) | 계정 비활성화 확인 모달 | 기본 / 호버 / 활성 상태일 때만 표시 |
| 활성화 버튼 | Button (secondary) | 계정 활성화 | 비활성 상태일 때만 표시 |
| 삭제 버튼 | Button (danger) | 계정 삭제 확인 모달 | 기본 / 호버 |

### 프로필 카드

| 부가 정보 | Text (secondary) | 가입일 · 최근 접속 | - |

### 탭 바

## 인터랙션 상세

### 역할 관리

| # | 사용자 액션 | 시스템 반응 |
|---|-------------|-------------|
| 1 | 역할 추가 드롭다운에서 역할 선택 → 추가 클릭 | API 호출 → 성공: 역할 목록에 추가, 최종 권한 갱신, 토스트: "역할이 할당되었습니다" |
| 2 | 현재 역할의 제거 클릭 | API 호출 → 성공: 역할 목록에서 제거, 최종 권한 갱신, 토스트: "역할이 해제되었습니다" |
| 3 | 마지막 역할 제거 시도 | 제거 버튼 비활성, 툴팁: "최소 1개의 역할이 필요합니다" |
| 4 | OWNER 역할 제거 시도 | 제거 버튼 비활성, 툴팁: "OWNER 역할은 해제할 수 없습니다" |

### 계정 관리

| # | 사용자 액션 | 시스템 반응 |
|---|-------------|-------------|
| 5 | 비활성화 클릭 | 비활성화 확인 모달 표시 |
| 6 | 비활성화 확인 | 계정 비활성화, 상태 배지 🔴 변경, 토스트: "계정이 비활성화되었습니다", 버튼 "활성화"로 변경 |
| 7 | 활성화 클릭 | 즉시 활성화 (확인 모달 없음), 상태 배지 🟢 변경, 토스트: "계정이 활성화되었습니다" |
| 8 | 삭제 클릭 | 삭제 확인 모달 표시 (데이터 처리 옵션 포함) |
| 9 | 삭제 확인 (파일 삭제 선택) | 계정 + 파일 영구 삭제 → A-03으로 이동, 토스트: "계정이 삭제되었습니다" |
| 10 | 삭제 확인 (파일 보존 선택) | 계정 삭제 + 파일을 관리자에게 이관 → A-03으로 이동, 토스트: "계정이 삭제되었습니다. 파일은 관리자에게 이관되었습니다" |

### 활동 조회

| # | 사용자 액션 | 시스템 반응 |
|---|-------------|-------------|
| 11 | 활동 탭 클릭 | 해당 사용자의 최근 활동 20건 로드 |
| 12 | 더 보기 클릭 | 추가 20건 로드 (append) |

## 모달/오버레이 상세

### 비활성화 확인 모달

```
┌──────────────────────────────────────────┐
│ 계정 비활성화                        ✕   │
│                                          │
│ "지은" 계정을 비활성화하시겠습니까?      │
│                                          │
│ 비활성화된 계정은 로그인할 수 없으며,    │
│ 기존 파일과 데이터는 유지됩니다.        │
│                                          │
│                    [취소]  [비활성화]     │
└──────────────────────────────────────────┘
```

### 삭제 확인 모달

```
┌──────────────────────────────────────────┐
│ 계정 삭제                            ✕   │
│                                          │
│ ⚠ "지은" 계정을 삭제하시겠습니까?        │
│                                          │
│ 이 작업은 취소할 수 없습니다.            │
│                                          │
│ 파일 처리 방법을 선택하세요:             │
│ ○ 파일도 함께 삭제 (892개 파일, 18.7 GB) │
│ ● 파일을 관리자에게 이관                 │
│                                          │
│                    [취소]  [삭제]         │
└──────────────────────────────────────────┘
```

- 삭제 버튼: danger 스타일 (빨간색)
- 파일 처리 옵션: 라디오 버튼, 기본값 "파일을 관리자에게 이관" (안전한 옵션)
- 파일 삭제 선택 시 파일 수와 용량 표시로 영향도 시각화

## 접근성

- 탭 바: `role="tablist"` + `role="tab"`, `aria-selected`
- 역할 제거 버튼: `aria-label` (역할명 포함, 예: "USER 역할 해제")
- 프로그레스 바: `role="progressbar"`, `aria-valuenow`, `aria-valuemax`
- 삭제 모달: `role="alertdialog"` (경고성 대화상자)
- 비활성화 모달: `role="dialog"`
- 상태 배지: 색상 외에 텍스트로 상태 전달
