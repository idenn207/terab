# D-01b 로그인 — 백업 코드 인증 (PC Web)

## 화면 개요

- 목적: Push 2FA 불가 시 백업 코드로 인증
- 진입 경로: D-01a에서 "백업 코드로 인증" 클릭
- 사용자: 디바이스 오프라인/분실 사용자
- 레이아웃: **Layout-C** (인증 페이지)
- 플랫폼: **PC Web 전용**

## 레이아웃 구조

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│              ┌──────────────────────┐                    │
│              │    🗂 NAS Drive      │                    │
│              │                      │                    │
│              │   백업 코드 입력      │                    │
│              │                      │                    │
│              │  ┌──────────────────┐│                    │
│              │  │                  ││                    │
│              │  └──────────────────┘│                    │
│              │                      │                    │
│              │  ┌──────────────────┐│                    │
│              │  │     인증하기     ││                    │
│              │  └──────────────────┘│                    │
│              │                      │                    │
│              │  ← Push 인증으로 돌아가기               │                    │
│              └──────────────────────┘                    │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

## 구성 요소

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 안내 문구 | Text | "백업 코드 입력" | - |
| 백업 코드 입력 | Input (text) | 8자리 백업 코드 (예: ABCD-1234) | 기본 / 포커스 / 에러 |
| 인증하기 버튼 | Button (primary) | 백업 코드 검증 API 호출 | 기본 / 호버 / 로딩 / 비활성 |
| 돌아가기 링크 | TextLink | D-01a로 복귀 | 기본 / 호버 |
| 에러 메시지 | Alert | "유효하지 않은 백업 코드입니다" | 에러 |
| 잔여 코드 안내 | Text (warning) | 인증 성공 후: "백업 코드가 N개 남았습니다" | 경고(3개 이하) |

## 인터랙션

| 사용자 액션 | 시스템 반응 |
|-------------|-------------|
| 백업 코드 입력 후 인증 클릭 | 코드 검증 → 성공 시 JWT 발급 → `/drive` 이동. 해당 코드 소진 처리 |
| 잘못된 코드 입력 | 에러 메시지 표시 |
| 인증 성공 + 코드 3개 이하 | 로그인 후 "새 디바이스 등록" 안내 배너 표시 |
| 돌아가기 클릭 | D-01a (Push 2FA 대기) 화면 복귀 |
