# A-08 스토리지 대시보드

## 화면 개요

- 목적: 전체 스토리지 용량과 사용자별 점유 현황을 상세 조회
- 진입 경로: `/storage` (사이드바 "스토리지" 클릭, 또는 A-02 대시보드에서 이동)
- 사용자: `storage:manage` 권한 보유 관리자 (페르소나: 민수)
- 레이아웃: **Layout-B** (Admin Web 기본)
- 플랫폼: PC Web 전용
- 관련 요구사항: ADMIN-06, ADMIN-07(Should)

## 구성 요소

### Page Header

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 페이지 제목 | Text (h1) | "💾 스토리지" | - |

### 전체 용량 요약

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 도넛 차트 | Chart (donut) | 사용량/여유량 비율 시각화, 중앙에 백분율 표시 | 정상(파란색) / 경고(80%+, 노란색) / 위험(90%+, 빨간색) |
| 전체 용량 | Text | 총 스토리지 용량 (MinIO 설정 기준) | - |
| 사용량 | Text | 현재 사용량 + 백분율 | - |
| 여유량 | Text | 남은 용량 + 백분율 | - |
| 사용자 수 | Text | 파일을 보유한 사용자 수 | - |

### 사용자별 점유 현황

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 검색 입력 | Input (search) | 사용자 이름으로 필터링 | 기본 / 포커스 |
| 정렬 드롭다운 | Select | 사용량(기본 내림차순) / 사용률 / 이름 / 파일 수 | - |
| 테이블 헤더 | TableHeader | 이름 · 사용량 · 할당량 · 사용률 · 파일 수 | 정렬 가능 |
| 사용자 행 | TableRow | 이름 + 사용량 + 할당량 + 프로그레스 바 + 파일 수 | 기본 / 호버 |
| 사용률 프로그레스 바 | ProgressBar | 사용률 시각화 | 정상(초록) / 경고(80%+, 노란) / 위험(90%+, 빨간) |
| 페이지네이션 | Pagination | 20건/페이지, 이전/다음 버튼 | - |

## 인터랙션 상세

| # | 사용자 액션 | 시스템 반응 |
|---|-------------|-------------|
| 1 | 페이지 진입 | 전체 용량 요약 + 사용자별 점유 데이터 로드 |
| 2 | 사용자 행 클릭 | A-04(사용자 상세)로 이동 |
| 3 | 검색 입력 | 사용자 이름으로 테이블 실시간 필터링 (디바운스 300ms) |
| 4 | 정렬 변경 | 선택한 기준으로 테이블 재정렬 |
| 5 | 페이지네이션 클릭 | 해당 페이지 데이터 로드 |

## 반응형 동작

| 브레이크포인트 | 변경 사항 |
|----------------|-----------|
| Desktop (1024px+) | Sidebar 240px + 도넛 차트 좌측 + 수치 우측 인라인 |
| Tablet (768~1023px) | Sidebar 접힘, 도넛 차트와 수치 세로 스택 |
| Mobile (< 768px) | Sidebar 숨김, "파일 수" 컬럼 숨김, 테이블 가로 스크롤 |

## 접근성

- 도넛 차트: `role="img"`, `aria-label` (사용률 수치 포함)
- 프로그레스 바: `role="progressbar"`, `aria-valuenow`, `aria-valuemax`, `aria-label`
- 테이블: `role="table"`, 정렬 상태 `aria-sort`
- 검색 입력: `<label>` 연결, `aria-describedby`
- 위험 수준(90%+) 프로그레스 바: `aria-live="polite"` 상태 변경 알림
