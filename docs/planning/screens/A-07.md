# A-07 역할 생성/수정

## 화면 개요

- 목적: 커스텀 역할 생성 또는 기존 역할의 이름/설명/권한 수정
- 진입 경로: `/roles/new` (역할 생성) 또는 `/roles/:id` (역할 수정, A-06에서 편집 클릭)
- 사용자: `user:role` 권한 보유 관리자 (페르소나: 민수)
- 레이아웃: **Layout-B** (Admin Web 기본)
- 플랫폼: PC Web 전용
- 관련 요구사항: ADMIN-17
- 관련 유저플로우: Flow 6 — 역할 생성/편집

## 구성 요소

### Page Header

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 뒤로가기 | Link | "← 역할 목록" → A-06으로 이동 | 기본 |
| 페이지 제목 | Text (h1) | 생성 모드: "역할 생성" / 수정 모드: "역할 수정" | - |

### 역할 정보

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 역할 이름 | Input | 역할 이름 입력 (필수, 대문자 권장) | 기본 / 포커스 / 에러 |
| 역할 설명 | Textarea | 역할 목적 설명 (선택) | 기본 / 포커스 |

### 권한 설정

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 전체 선택 체크박스 | Checkbox | 모든 권한 선택/해제 | 미체크 / 체크 / 부분 체크(indeterminate) |
| 리소스 그룹 헤더 | Collapsible | 리소스 카테고리 (접기/펼치기), 그룹 체크박스 포함 | 펼침(기본) / 접힘 |
| 그룹 체크박스 | Checkbox | 해당 그룹 내 모든 권한 선택/해제 | 미체크 / 체크 / 부분 체크 |
| 권한 체크박스 | Checkbox | 개별 권한 (`resource:action`) + 한글 설명 | 미체크 / 체크 |

## 접근성

- 폼 필드: `<label>` 연결, `aria-required` (역할 이름)
- 체크박스 그룹: `role="group"`, `aria-labelledby` (그룹 헤더)
- 부분 선택(indeterminate): `aria-checked="mixed"`
- 접기/펼치기: `aria-expanded`, `aria-controls`
- 권한 체크박스: aria-label에 한글 설명 포함 (예: "user:read — 사용자 목록 조회")
- 하단 액션 바: 키보드 Tab 순서 보장
