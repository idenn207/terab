# D-01a 로그인 — 2단계: Push 2FA 대기 (PC Web)

## 화면 개요

- 목적: Push 2FA 승인 대기 (PC 브라우저에서 모바일 앱 승인을 기다리는 화면)
- 진입 경로: D-01에서 1차 인증 성공 후 자동 전환
- 사용자: 비신뢰기기에서 로그인하는 사용자
- 레이아웃: **Layout-C** (인증 페이지)
- 플랫폼: **PC Web 전용**

## 구성 요소

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 안내 문구 | Text | "모바일에서 승인하세요" | - |
| 챌린지 숫자 | Display (large) | 2자리 숫자 (예: 42), 큰 폰트로 강조 | 표시 중 |
| 안내 설명 | Text (secondary) | "표시된 숫자를 앱에서 입력하세요" | - |
| 대기 애니메이션 | Spinner/Dots | 승인 대기 중 시각적 피드백 | 대기 중 / 승인됨 / 거부됨 / 타임아웃 |
| 남은 시간 | Timer | 60초 카운트다운 | 진행 중 / 만료 |
| Push 미도착 안내 | Alert (warning) | Push 전달 실패 시 표시 | "알림이 전달되지 않았습니다. 앱을 직접 열어주세요" |
| 다시 보내기 | TextLink | Push 재전송 요청 | 기본 / 쿨다운(10초) |
| 백업 코드로 인증 | TextLink | 백업 코드 입력 화면으로 전환 | 기본 / 호버 |

## 인터랙션

| 사용자 액션 | 시스템 반응 |
|-------------|-------------|
| 화면 진입 | WebSocket/SSE 연결, 서버가 모바일에 Push 전송 |
| 모바일에서 승인 완료 | WebSocket으로 JWT 수신 → `/drive`로 자동 이동 |
| 모바일에서 거부 | "로그인이 거부되었습니다" 표시 → D-01로 복귀 |
| 60초 타임아웃 | "인증 시간이 만료되었습니다" 표시, 다시 보내기 활성화 |
| 다시 보내기 클릭 | 새 챌린지 생성 + Push 재전송, 숫자 변경, 타이머 리셋 |
| 백업 코드로 인증 클릭 | **D-01b (백업 코드 입력)** 화면으로 전환 |
| Push 미도착 (pushDelivered: false) | 경고 배너 표시: "앱을 직접 열어주세요" |
