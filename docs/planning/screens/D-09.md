# D-09 휴지통

## 화면 개요

- 목적: 삭제된 파일/폴더 목록 조회, 복원 또는 영구 삭제
- 진입 경로: `/trash` (사이드바 "휴지통" 클릭)
- 사용자: 로그인된 모든 사용자 (페르소나: 민수, 지은)
- 레이아웃: **Layout-A** (Drive Web 기본)
- 플랫폼: Web + 모바일 앱 공용
- 관련 요구사항: TRASH-01, TRASH-02

## 레이아웃 구조

```
┌─ Toolbar ──────────────────────────────────────────────────┐
│ 🗑 휴지통                        정렬▼  ≡    [휴지통 비우기] │
└────────────────────────────────────────────────────────────┘
┌─ 안내 배너 ────────────────────────────────────────────────┐
│ ℹ 휴지통의 항목은 30일 후 자동으로 영구 삭제됩니다          │
└────────────────────────────────────────────────────────────┘
┌─ Content Area ─────────────────────────────────────────────┐
│ ☐  이름              원래 위치          삭제일    남은일  ⋮ │
│ ──────────────────────────────────────────────────────────│
│ ☐ 📄 old_memo.txt   내 드라이브        3월 20일  29일   ⋮ │
│ ☐ 📁 백업/          내 드라이브 > 문서  3월 18일  27일   ⋮ │
│ ☐ 🖼 screenshot.png 내 드라이브 > 사진  3월 10일  19일   ⋮ │
│ ☐ 📄 draft.docx     내 드라이브 > 문서  3월 1일   10일   ⋮ │
└───────────────────────────────────────────────────────────┘
```

**선택 시 Selection Toolbar:**

```
┌─ Selection Toolbar ────────────────────────────────────────┐
│ ✕  2개 선택됨               [↩ 복원] [🗑 영구 삭제]        │
└────────────────────────────────────────────────────────────┘
```

## 구성 요소

### Toolbar 영역

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 페이지 제목 | Text (h1) | "🗑 휴지통" | - |
| 정렬 드롭다운 | Select | 삭제일 / 이름 / 크기 / 남은 일수 | 기본(삭제일 내림차순) |
| 보기 모드 | Text | **리스트 뷰 전용** (그리드 뷰 비활성) | - |
| 휴지통 비우기 | Button (danger) | 모든 항목 영구 삭제 확인 모달 열기 | 기본 / 비활성(빈 휴지통) |

### 안내 배너

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 자동 정리 안내 | InfoBanner | "휴지통의 항목은 30일 후 자동으로 영구 삭제됩니다" | 항상 표시, 닫기 불가 |

### Content Area — 리스트 뷰

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 테이블 헤더 | TableHeader | 전체 선택 체크박스 · 이름 · 원래 위치 · 삭제일 · 남은 일수 · 액션 | 정렬 가능 |
| 삭제된 항목 행 | TableRow | 체크박스 + 아이콘 + 이름 + 원래 위치 + 삭제일 + 남은 일수 + ⋮ | 기본 / 호버 / 선택 |
| 남은 일수 | Badge | 자동 삭제까지 남은 일수 | 정상(초록) / 임박(7일 이내, 노란색) / 위험(3일 이내, 빨간색) |

### Selection Toolbar

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 선택 해제 | IconButton (✕) | 선택 해제 | 기본 |
| 선택 카운트 | Text | "N개 선택됨" | - |
| 복원 | Button (primary) | 선택 항목 원래 위치로 복원 | 기본 |
| 영구 삭제 | Button (danger) | 선택 항목 영구 삭제 확인 모달 | 기본 |

### 컨텍스트 메뉴 (⋮ 클릭)

```
┌──────────────────────┐
│ ↩ 복원               │
│ ──────────────────── │
│ 🗑 영구 삭제          │
└──────────────────────┘
```

## 인터랙션 상세

### 복원

| # | 사용자 액션 | 시스템 반응 |
|---|-------------|-------------|
| 1 | 컨텍스트 메뉴 "복원" 또는 Selection Toolbar "복원" | 원래 위치로 파일 복원, 토스트: "N개 항목이 복원되었습니다" + [위치로 이동] |
| 2 | 토스트 "위치로 이동" 클릭 | 복원된 파일의 원래 폴더로 이동 |
| 3 | 원래 폴더가 삭제된 경우 | 상위 폴더 중 존재하는 가장 가까운 폴더로 복원, 토스트에 안내 |

### 영구 삭제

| # | 사용자 액션 | 시스템 반응 |
|---|-------------|-------------|
| 4 | 컨텍스트 메뉴 "영구 삭제" | 영구 삭제 확인 모달 표시 |
| 5 | 모달에서 "영구 삭제" 클릭 | 파일 완전 삭제 (복구 불가), 목록에서 제거, 사이드바 스토리지 바 갱신 |

### 전체 비우기

| # | 사용자 액션 | 시스템 반응 |
|---|-------------|-------------|
| 6 | "휴지통 비우기" 클릭 | 전체 비우기 확인 모달 표시 |
| 7 | 모달에서 "비우기" 클릭 | 모든 항목 영구 삭제, 빈 휴지통 상태로 전환 |

### 멀티셀렉트

| # | 사용자 액션 | 시스템 반응 |
|---|-------------|-------------|
| 8 | Ctrl/Shift + 클릭, 체크박스 | D-03과 동일한 멀티셀렉트 동작 |
| 9 | 헤더 전체 선택 체크박스 | 전체 선택/해제 |

## 모달/오버레이 상세

### 영구 삭제 확인 모달

```
┌──────────────────────────────────────────┐
│ 영구 삭제                            ✕   │
│                                          │
│ ⚠ "old_memo.txt"을(를) 영구 삭제         │
│ 하시겠습니까?                            │
│                                          │
│ 이 작업은 취소할 수 없으며,              │
│ 파일은 완전히 삭제됩니다.                │
│                                          │
│                    [취소]  [영구 삭제]    │
└──────────────────────────────────────────┘
```

- 영구 삭제 버튼: danger 스타일 (빨간색)
- 다중 선택 시: "3개 항목을 영구 삭제하시겠습니까?"
- 폴더 삭제 시: "폴더 내 모든 파일이 함께 영구 삭제됩니다"

### 전체 비우기 확인 모달

```
┌──────────────────────────────────────────┐
│ 휴지통 비우기                        ✕   │
│                                          │
│ ⚠ 휴지통의 모든 항목(N개)을 영구         │
│ 삭제하시겠습니까?                        │
│                                          │
│ 이 작업은 취소할 수 없습니다.            │
│                                          │
│                    [취소]  [비우기]       │
└──────────────────────────────────────────┘
```

- 비우기 버튼: danger 스타일

## 반응형 동작

| 브레이크포인트 | 변경 사항 |
|----------------|-----------|
| Desktop (1024px+) | Sidebar 240px + 리스트 전체 컬럼 표시 |
| Tablet (768~1023px) | Sidebar 접힘, "남은 일수" 컬럼 숨김 (삭제일만 표시) |
| Mobile (< 768px) | Sidebar 완전 숨김, "원래 위치" 컬럼 숨김, 컨텍스트 메뉴 대신 바텀 시트 |

## 접근성

- 삭제 항목 리스트: `role="table"`, 각 행에 aria-label (이름 + 삭제일 + 남은 일수)
- 남은 일수 배지: `role="status"`, 위험(3일 이내) 시 `aria-live="polite"` 업데이트
- 영구 삭제 모달: `role="alertdialog"` (경고성 대화상자)
- 안내 배너: `role="note"`
- 키보드: Delete 키 → 영구 삭제 확인 모달
