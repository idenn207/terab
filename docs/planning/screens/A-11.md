# A-11 서비스 상태

## 화면 개요

- 목적: API, MinIO, PostgreSQL 컨테이너의 Health 상태를 조회하고 이상 여부를 확인
- 진입 경로: `/system/services` (사이드바 "시스템" → 서브메뉴 "서비스 상태")
- 사용자: `system:monitor` 권한 보유 관리자 (페르소나: 민수)
- 레이아웃: **Layout-B** (Admin Web 기본)
- 플랫폼: PC Web 전용
- 관련 요구사항: ADMIN-12

## 인터랙션 상세

| # | 사용자 액션 | 시스템 반응 |
|---|-------------|-------------|
| 1 | 페이지 진입 | 3개 서비스 Health Check 실행 + 상태 변경 이력 로드 + 30초 자동 갱신 시작 |
| 2 | "상태 확인" 클릭 | 즉시 Health Check 실행, 버튼 로딩 상태, 30초 타이머 리셋 |
| 3 | 30초 경과 | 자동 Health Check 실행, 마지막 확인 시각 갱신 |
| 4 | 상태 변경 감지 | 카드 상태 업데이트 + 이력 테이블 최상단에 신규 항목 추가 |
| 5 | "더보기" 클릭 | 다음 10건 이력 로드 (누적 표시) |
| 6 | 브라우저 탭 비활성화 | 자동 갱신 일시정지 |
| 7 | 브라우저 탭 다시 활성화 | 즉시 Health Check + 자동 갱신 재시작 |

### Health Check 동작

- 각 서비스의 Health 엔드포인트 호출:
  - API Server: `/actuator/health`
  - MinIO: MinIO 관리 API Health 엔드포인트
  - PostgreSQL: `SELECT 1` 또는 JDBC 연결 확인
- 타임아웃: 5초 (5초 내 미응답 시 🔴 중단 판정)
- 응답 시간 기준: < 200ms (정상), 200~500ms (응답 지연), > 500ms 또는 실패 (중단)

## 접근성

- 서비스 카드: `role="status"`, `aria-label` (예: "API Server 상태: 정상, 응답 시간 23ms")
- 상태 표시등: 색상 외 텍스트 라벨 병행 ("정상", "응답 지연", "중단")
- 상태 변경 시: `aria-live="polite"` (변경된 서비스 상태 음성 안내)
- "상태 확인" 버튼: 확인 완료 시 `aria-live="polite"`로 결과 안내
- 이력 테이블: `role="table"`, 시각순 정렬
- 위험 배너 (전체 이상 시): `role="alert"`
