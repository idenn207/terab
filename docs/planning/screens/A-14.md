# A-14 감사 로그

## 화면 개요

- 목적: 서비스 내 모든 사용자 행위를 기록한 감사 로그를 조회하고 분석
- 진입 경로: `/audit/logs` (사이드바 "감사 로그" 클릭)
- 사용자: `audit:read` 권한 보유 관리자 (페르소나: 민수)
- 레이아웃: **Layout-B** (Admin Web 기본)
- 플랫폼: PC Web 전용
- 관련 요구사항: ADMIN-15

│ 📅 [03-15] ~ [03-22]   사용자: [전체 ▾]                     │

## 구성 요소

### Page Header

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 페이지 제목 | Text (h1) | "📋 감사 로그" | - |
| CSV 내보내기 버튼 | Button (secondary) | 📥 현재 필터 기준으로 CSV 다운로드 | 기본 / 로딩(생성 중) / 비활성(결과 없을 때) |

### 필터 바

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 날짜 범위 | DateRangePicker | 시작일 ~ 종료일 (기본: 최근 7일) | 기본 / 포커스 / 에러(시작일 > 종료일) |
| 사용자 필터 | Select | 전체 / 특정 사용자 선택 | 기본 |
| 활동 유형 필터 | Multi-select 드롭다운 | 로그인, 파일 업로드, 파일 삭제, 공유, 설정 변경, 사용자 관리 | 기본: 전체 선택 |
| 검색 입력 | Input (search) | 대상 파일명/설명 텍스트 검색 | 기본 / 포커스 |

### 로그 테이블

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 테이블 헤더 | TableHeader | 시각 · 사용자 · 활동 유형 · 대상 · IP 주소 | 시각 기본 내림차순 |
| 시각 | TableCell | "MM-DD HH:mm:ss" 형식 | - |
| 사용자 | TableCell | 사용자 이름 | 클릭 시 A-04 이동 |
| 활동 유형 배지 | Badge | 유형별 색상 배지 (아래 색상 참조) | - |
| 대상 | TableCell | 파일/폴더명, 설정 항목명 등. 없을 경우 "—" | - |
| IP 주소 | TableCell | 접근 IP (축약 표시, 호버 시 전체 표시) | 호버 시 툴팁 |
| 행 | TableRow | 클릭 시 상세 모달 표시 | 기본 / 호버 / 선택(모달 표시 중) |
| 커서 페이지네이션 | Navigation | 이전/다음 버튼만 표시 (페이지 번호 없음) | 이전 비활성(첫 페이지) / 다음 비활성(마지막) |

### 커서 기반 페이지네이션

> 감사 로그는 대량 데이터가 누적되므로 오프셋 기반 대신 **커서 기반** 페이지네이션을 사용합니다.
> - 페이지 번호나 전체 건수를 표시하지 않음
> - "이전 / 다음" 버튼만 제공
> - 내부적으로 마지막 조회 항목의 timestamp를 커서로 사용
> - 장점: 대량 데이터에서도 일정한 조회 성능 유지

## 접근성

- 테이블: `role="table"`, 정렬 상태 `aria-sort`
- 활동 유형 배지: 색상 외 텍스트 라벨 병행
- DateRangePicker: `aria-label="조회 기간"`, 시작일/종료일 각각 `<label>` 연결
- Multi-select 드롭다운: `aria-label="활동 유형 필터"`, 선택 상태 `aria-checked`
- 모달: 포커스 트랩, Esc로 닫기, `role="dialog"`, `aria-labelledby`
- 필터 변경 시 결과 업데이트: `aria-live="polite"`
- CSV 내보내기 완료: `aria-live="polite"` 안내
