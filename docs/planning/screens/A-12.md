# A-12 회원가입 설정

## 화면 개요

- 목적: 서비스 회원가입 정책 설정 (초대 필수 여부, 최대 사용자 수 제한)
- 진입 경로: `/settings/registration` (사이드바 "설정" → 서브메뉴 "회원가입")
- 사용자: `system:config` 권한 보유 관리자 (페르소나: 민수)
- 레이아웃: **Layout-B** (Admin Web 기본)
- 플랫폼: PC Web 전용
- 관련 요구사항: ADMIN-13

## 인터랙션 상세

| # | 사용자 액션 | 시스템 반응 |
|---|-------------|-------------|
| 1 | 페이지 진입 | 현재 설정값 로드 (초대 필수 여부, 최대 사용자 수) + 현재 등록 사용자 수 표시 |
| 2 | 토글 전환 | 토글 상태 변경, 취소/저장 버튼 활성화 |
| 3 | 최대 사용자 수 변경 | 입력값 변경, 취소/저장 버튼 활성화 |
| 4 | 저장 클릭 | 유효성 검증 → 성공 시 API 호출 → 성공 Toast "회원가입 설정이 저장되었습니다" |
| 5 | 취소 클릭 | 모든 필드를 서버에서 로드한 초기값으로 복원, 취소/저장 버튼 비활성화 |
| 6 | 변경 사항 있는 상태에서 페이지 이탈 | "저장하지 않은 변경 사항이 있습니다. 이동하시겠습니까?" 확인 다이얼로그 |

### 유효성 검증

| 필드 | 규칙 | 에러 메시지 |
|------|------|-------------|
| 최대 사용자 수 | 0 이상의 정수 | "0 이상의 숫자를 입력해주세요" |
| 최대 사용자 수 | 0이 아닌 경우, 현재 등록 사용자 수 이상 | "현재 등록된 사용자(N명)보다 작게 설정할 수 없습니다" |

## 상태 목록

| # | 상태 | 설명 |
|---|------|------|
| 1 | 기본 | 현재 설정값 표시, 취소/저장 버튼 비활성 |
| 2 | 로딩 | 설정값 로드 중 — 토글 스켈레톤 + 입력 스켈레톤 |
| 3 | 변경됨 | 설정값이 초기값과 다름 — 취소/저장 버튼 활성화 |
| 4 | 저장 중 | 저장 API 호출 중 — 저장 버튼 로딩 스피너, 폼 비활성화 |
| 5 | 에러 | "설정을 불러올 수 없습니다" + [다시 시도] |

## 반응형 동작

| 브레이크포인트 | 변경 사항 |
|----------------|-----------|
| Desktop (1024px+) | Sidebar 240px + 설정 폼 카드 max-width 640px 중앙 배치 |
| Tablet (768~1023px) | Sidebar 접힘, 폼 카드 전체 너비 |
| Mobile (< 768px) | Sidebar 숨김, 폼 카드 전체 너비, 하단 액션 바 고정(sticky) |

## 접근성

- 토글 스위치: `role="switch"`, `aria-checked`, `aria-label="초대 필수 여부"`
- 숫자 입력: `<label>` 연결, `aria-describedby` (도움말 텍스트 연결)
- 에러 메시지: `role="alert"`, `aria-live="assertive"`
- 저장 성공 Toast: `aria-live="polite"`
- 이탈 확인 다이얼로그: 포커스 트랩, Esc로 취소
