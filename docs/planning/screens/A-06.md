# A-06 역할 목록

## 화면 개요

- 목적: 서비스에 정의된 역할 목록 조회, 커스텀 역할 생성/편집/삭제 진입
- 진입 경로: `/roles` (사이드바 "역할/권한" 클릭)
- 사용자: `user:role` 권한 보유 관리자 (페르소나: 민수)
- 레이아웃: **Layout-B** (Admin Web 기본)
- 플랫폼: PC Web 전용
- 관련 요구사항: ADMIN-17, ADMIN-18
- 관련 유저플로우: Flow 6 — 역할/권한 관리

## 구성 요소

### Page Header

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 페이지 제목 | Text (h1) | "🛡 역할/권한 관리" | - |
| 역할 생성 버튼 | Button (primary) | A-07(역할 생성 `/roles/new`)으로 이동 | 기본 / 호버 |

### 역할 카드

| 요소 | 타입 | 설명 | 상태 |
|------|------|------|------|
| 역할 아이콘 | Icon | 역할 유형별 아이콘 (OWNER: 👑, ADMIN: 🔧, USER: 👤, 커스텀: 📋) | - |
| 역할명 | Text (h3) | 역할 이름 (대문자) | - |
| 역할 유형 배지 | Badge | "시스템 역할" (기본 역할) 또는 "커스텀 역할" | 시스템(회색+🔒) / 커스텀(초록) |
| 역할 설명 | Text (secondary) | 역할 목적 한 줄 설명 | - |
| 권한 수 | Text | "권한 N개" | - |
| 사용자 수 | Text | "사용자 N명" (클릭 시 해당 역할의 사용자 목록 필터링) | 클릭 가능 |
| 편집 버튼 | Button (secondary) | A-07(역할 수정 `/roles/:id`)으로 이동 | 커스텀 역할에만 표시 |
| 삭제 버튼 | Button (danger, text) | 역할 삭제 확인 모달 | 커스텀 역할에만 표시 |

## 인터랙션 상세

| # | 사용자 액션 | 시스템 반응 |
|---|-------------|-------------|
| 1 | 역할 생성 클릭 | A-07(역할 생성)으로 이동 |
| 2 | 커스텀 역할 편집 클릭 | A-07(역할 수정 `/roles/:id`)으로 이동 |
| 3 | 커스텀 역할 삭제 클릭 (사용자 없음) | 삭제 확인 모달 → 확인 시 삭제, 토스트: "역할이 삭제되었습니다" |
| 4 | 커스텀 역할 삭제 클릭 (사용자 있음) | 대체 역할 선택 모달 → 대체 역할 선택 후 삭제, 토스트: "역할이 삭제되었습니다. N명의 사용자에게 [대체역할]이 할당되었습니다" |
| 5 | 사용자 수 클릭 | A-03(사용자 목록)으로 이동 (해당 역할 필터 적용) |
| 6 | 시스템 역할 카드 클릭 | 권한 상세 표시 (확장 또는 A-07 읽기 전용 모드) |

| Tablet (768~1023px) | Sidebar 접힘, 카드 전체 너비 |

## 접근성

- 역할 카드: `role="article"`, 각 카드에 aria-label (역할명 + 유형)
- 시스템 역할 잠금 아이콘: `aria-label="편집 불가"`, 스크린리더에 "시스템 역할, 편집 불가" 안내
- 삭제 모달: `role="alertdialog"`
- 사용자 수 링크: `role="link"`, 명확한 aria-label
