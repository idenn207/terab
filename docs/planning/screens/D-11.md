# D-11 공유 링크 페이지

## 화면 개요

- 목적: 공유 링크를 통해 비로그인 사용자가 파일/폴더에 접근하고 다운로드
- 진입 경로: `/s/:shareId` (공유 링크 URL 직접 접근)
- 사용자: **비로그인 사용자 포함** (공유 링크를 받은 누구나)
- 레이아웃: **Layout-D** (공유 링크 독립 레이아웃)
- 플랫폼: Web (모바일 브라우저 포함)
- 관련 요구사항: SHARE-01, SHARE-02
- 관련 유저플로우: Flow 5 "공유 링크 생성 및 접근" — 링크 접근 파트

## 인터랙션 상세

### 비밀번호 입력

| # | 사용자 액션 | 시스템 반응 |
|---|-------------|-------------|
| 1 | 페이지 접근 (비밀번호 보호 링크) | 비밀번호 입력 카드 표시 |
| 2 | 비밀번호 입력 → 확인 또는 Enter | API 검증 → 성공: 콘텐츠 카드로 전환 |
| 3 | 비밀번호 불일치 | 인라인 에러: "비밀번호가 일치하지 않습니다", 입력 필드 유지 |
| 4 | 5회 연속 실패 | "잠시 후 다시 시도해 주세요" (30초 대기), 확인 버튼 비활성 |

### 파일 다운로드

| # | 사용자 액션 | 시스템 반응 |
|---|-------------|-------------|
| 5 | 다운로드 버튼 클릭 (단일 파일) | 브라우저 기본 다운로드 시작 |
| 6 | 전체 다운로드 클릭 (폴더) | ZIP 생성 → 다운로드 (대용량 시 진행률 표시) |
| 7 | 개별 파일 클릭 (폴더 목록 내) | 해당 파일만 다운로드 |

### 폴더 탐색 (공유 범위 내)

| # | 사용자 액션 | 시스템 반응 |
|---|-------------|-------------|
| 8 | 하위 폴더 클릭 | 공유 범위 내에서 하위 폴더 콘텐츠 표시 (URL 변경 없음, 카드 내 갱신) |
| 9 | 브레드크럼 상위 경로 클릭 | 해당 경로의 콘텐츠로 복귀 |
| 10 | 공유 범위 외 이동 시도 | 차단 (공유된 폴더 경계 밖으로 이동 불가) |

### 오류 상태

| # | 사용자 액션 | 시스템 반응 |
|---|-------------|-------------|

| 12 | 존재하지 않는 링크 접근 | 오류 카드 표시: "이 공유 링크는 존재하지 않습니다" |
| 13 | 삭제된 원본 파일의 링크 접근 | 오류 카드 표시: "공유된 파일이 삭제되었습니다" |

## 반응형 동작

| 브레이크포인트 | 변경 사항 |
|----------------|-----------|
| Desktop (1024px+) | 콘텐츠 카드 max-width 800px, 중앙 정렬 |
| Tablet (768~1023px) | 콘텐츠 카드 좌우 여백 24px |
| Mobile (< 768px) | 콘텐츠 카드 전체 너비 (여백 16px), 파일 리스트 크기 컬럼 숨김, 다운로드 버튼 전체 너비 |

## 접근성

- 비밀번호 입력: `<label>` 연결, `aria-describedby` (에러 메시지)
- 오류 상태: `role="alert"`, 스크린리더 즉시 읽기
- 다운로드 버튼: 명확한 aria-label ("파일명 다운로드")
- 파일 리스트: `role="table"`, 각 행에 aria-label
- 전체 페이지: 사이드바/네비게이션 없으므로 랜드마크 구조 단순 (`main`, `header`, `footer`)
